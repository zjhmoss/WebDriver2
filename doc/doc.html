introduction

This automation project consists of WebDriver level 2 language bindings and a simple way to represent pages. It is implemented in Raku, formerly known as Perl 6.

page representation

Pages are outlined using a simple language. The outline is then transformed into a tree of objects that services can use to provide actions a user would be able to take on that page. The language is defined using the Raku grammar in PO/Build/Page.rakumod.



services

Services, which expose actions that can be taken for a given page or component within a page, have access to the page representation. Test scripts, which have no knowledge of the underlying page structure, then use the services to test the page ( component ) functionality.



scripting

Tests are written as Raku scripts using the standard Test module. Scripts specify the page outline and services they will be using and the project builds the page representation for them.
